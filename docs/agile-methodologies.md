# Agile Oyun Geliştirme Metodolojileri

Oyun sektöründe klasik Agile yöntemleri doğrudan kullanılmaz. Oyunların yaratıcı ve değişken yapısına göre uyarlanır.

## 1. Scrum 

**Nasıl Çalışır?**
- 1-4 haftalık sprintler
- Sprint Planning, Daily Standup, Sprint Review, Retrospective
- Product Backlog ve Sprint Backlog kullanılır

**Oyun Sektöründe Avantajı:**
- Yeni feature’ları kısa sürede test etme imkanı
- Oyuncu feedback’ini hızlı almak

**Dezavantajı:**
- Yaratıcı işlerde (sanat, oyun tasarımı) tam esneklik sağlamayabilir

## 2. Kanban

**Nasıl Çalışır?**
- Sürekli akış (sabit sprint yok)
- WIP (Work In Progress) limitleri
- Board’da kolonlar: To Do → In Progress → Review → Done

**Oyun Sektöründe En Çok Kullanıldığı Yer:**
- Live Ops (oyun çıktıktan sonraki dönem)
- Bug fix ve acil içerik üretimi

## 3. Scrumban 

Scrum + Kanban karışımıdır. 

**En Popüler Yaklaşım Nedeni:**
- Sprintlerle disiplin korunur
- Kanban ile esneklik sağlanır
- Özellikle orta ölçekli stüdyolarda tercih edilir

## 4. Diğer Yöntemler

- **Lean Development**: İsrafı azaltma, hızlı prototip
- **Feature Driven Development (FDD)**: Feature bazlı ilerleme

## Oyun Sektörüne Özgü Zorluklar

- Yaratıcı işlerin tahmin edilmesi zordur
- Scope creep (kapsam şişmesi) çok görülür
- Oyuncu feedback’i sürekli değişir
