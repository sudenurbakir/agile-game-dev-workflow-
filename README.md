# Oyun Stüdyosu için Agile Süreç Rehberi

Bu repo, oyun geliştirme projelerinde kullanılan **Scrum + Kanban hibrit** sürecini anlatır.

## Süreç Nasıl İşler?

Oyun geliştirme iki ana şekilde ilerler:

### 1. Sprintler (2 haftalık periyotlar)
- Belirli bir sürede belirli işler tamamlanır.
- Örnek: Yeni karakter ekleme, seviye tasarımı, hikaye yazımı.
- Her sprint sonunda neyin bittiği gözden geçirilir.

### 2. Kanban (Sürekli Akış)
- Sprint dışında gelen işler burada yönetilir.
- Özellikle oyun yayınlandıktan sonra (Live Ops) çok kullanılır.
- Örnek: Bug düzeltme, yeni event hazırlama, küçük güncellemeler.

## Live Ops Nedir?
Oyun çıktıktan sonraki dönemdir. Oyunun canlı kalması, oyuncu tutma ve yeni içerik üretme çalışmaları yapılır.

## Avantajı
- Hem planlı ilerliyoruz (Sprint)
- Hem de acil işlere hızlı cevap verebiliyoruz (Kanban)
